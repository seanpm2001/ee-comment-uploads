Control Panel

There is no settings or controls for Comment Uploads in the ExpressionEngine control panel. However, there's a few things you may be interested in knowing about:

  • File Upload Preference Creation
  • Deleting Entries and/or Comments

File Upload Preference Creation

Upon installation/enabling of the Comment Uploads extension, it attempts to create a folder called comments in the ./images/uploads directory (if it has not yet been created), as well as a new File Upload preference called Comment Uploads Directory. This becomes the default upload directory for all of your comment uploads, unless otherwise specified in your Comment Form.

Deleting Entries and/or Comments

Comment Uploads does NOT orphan uploaded files or data in your database. Comment Uploads uses the following processes:

  • Upon deletion of an individual comment, the related files are deleted from the server, and the related data is removed from the database.
  • Upon deletion of an individual entry, the related files of all comments are deleted from the server, and the related data is removed from the database.

NOTE: disabling of the Comment Uploads extension does NOT delete any of the uploaded files, but DOES delete all of the comment upload data from the database.
